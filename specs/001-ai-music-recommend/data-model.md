# Data Model: AI Music Recommendation via Vocal Analysis

**Feature**: 001-ai-music-recommend
**Date**: 2026-03-29

## Entities

### User

Represents a person using the system. Supports both anonymous and registered
modes.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| id | UUID | PK, auto-generated | Unique identifier |
| email | string | unique, nullable | Null for anonymous users |
| password_hash | string | nullable | Null for anonymous users |
| display_name | string | nullable | Optional display name |
| locale | enum | "en" \| "zh-CN" | UI language preference |
| created_at | datetime | not null | Account creation timestamp |
| updated_at | datetime | not null | Last profile update |

**Notes**:
- Anonymous users are tracked via browser localStorage on the client side
  and are NOT stored in this table.
- A user becomes "registered" when they provide email + password to enable
  the history/comparison feature (User Story 4, P3).

### AnalysisSession

A single end-to-end interaction from audio input to recommendations.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| id | UUID | PK, auto-generated | Unique identifier |
| user_id | UUID | FK → User, nullable | Null for anonymous sessions |
| source_type | enum | "recording" \| "upload" | How audio was provided |
| audio_duration_sec | float | not null, > 0 | Duration of analyzed audio segment |
| audio_format | string | nullable | Original format (mp3, wav, m4a, ogg) — only for uploads |
| signal_quality_score | float | 0.0–1.0 | Assessed quality of input signal |
| status | enum | "processing" \| "completed" \| "failed" | Analysis pipeline status |
| error_message | string | nullable | Reason for failure if status = failed |
| created_at | datetime | not null | Session start timestamp |
| completed_at | datetime | nullable | When analysis finished |

**Notes**:
- No raw audio is persisted (SC-007). The audio is processed in memory
  (recording path) or as a temporary file (upload path) and discarded
  after feature extraction.
- Anonymous sessions (user_id = null) are returned to the client and
  stored in browser localStorage for the single-session flow.

### VocalProfile

The AI-generated analysis result for one audio input. One per session.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| id | UUID | PK, auto-generated | Unique identifier |
| session_id | UUID | FK → AnalysisSession, unique | One profile per session |
| pitch_min_hz | float | not null | Lowest detected fundamental frequency |
| pitch_max_hz | float | not null | Highest detected fundamental frequency |
| pitch_min_note | string | not null | Musical note name (e.g., "C3") |
| pitch_max_note | string | not null | Musical note name (e.g., "G4") |
| pitch_median_hz | float | not null | Median pitch frequency |
| pitch_stability | float | 0.0–1.0 | How stable the pitch is over time |
| rhythm_tempo_bpm | float | nullable | Estimated tempo in BPM |
| rhythm_regularity | float | 0.0–1.0 | How regular the rhythm pattern is |
| mood_valence | float | 0.0–1.0 | Positive (1.0) vs negative (0.0) mood |
| mood_energy | float | 0.0–1.0 | High energy (1.0) vs calm (0.0) |
| mood_tension | float | 0.0–1.0 | Tense (1.0) vs relaxed (0.0) |
| mood_label | string | not null | Human-readable mood summary |
| timbre_warmth | float | 0.0–1.0 | Warm vs cold tonal quality |
| timbre_brightness | float | 0.0–1.0 | Bright vs dark spectral balance |
| timbre_breathiness | float | 0.0–1.0 | Breathy vs clear vocal quality |
| timbre_label | string | not null | Human-readable timbre summary |
| expression_vibrato | float | 0.0–1.0 | Vibrato presence and strength |
| expression_dynamic_range | float | 0.0–1.0 | Volume variation range |
| expression_articulation | string | not null | e.g., "legato", "staccato", "mixed" |
| confidence_overall | float | 0.0–1.0 | Overall confidence in the analysis |
| raw_features_json | JSON | not null | Full extracted feature vector for reproducibility |
| created_at | datetime | not null | Profile generation timestamp |

**Notes**:
- `raw_features_json` stores the complete feature vector (MFCCs, spectral
  features, pitch contour statistics) sent from the client. This enables
  re-running the recommendation engine without re-analyzing audio.
- Trait labels (mood_label, timbre_label) are plain-language descriptions
  generated by the analysis pipeline, as required by FR-004.

### Recommendation

A song direction suggestion derived from a vocal profile.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| id | UUID | PK, auto-generated | Unique identifier |
| session_id | UUID | FK → AnalysisSession | Parent session |
| rank | integer | not null, 1-8 | Display order |
| genre | string | not null | Primary genre/style label |
| sub_style | string | nullable | More specific style descriptor |
| tempo_range_low | integer | not null | Suggested minimum BPM |
| tempo_range_high | integer | not null | Suggested maximum BPM |
| vocal_difficulty | integer | 1–5 | Vocal skill level required |
| mood_alignment | string | not null | How this direction matches user's mood |
| match_explanation | string | not null | 1-2 sentence personalized explanation |
| confidence | enum | "high" \| "medium" \| "exploratory" | Match confidence level |
| reference_songs | JSON | nullable | Array of example song names (illustrative) |
| match_score | float | 0.0–1.0 | Rule engine match score |
| created_at | datetime | not null | Recommendation generation timestamp |

**Notes**:
- 3–8 recommendations per session as required by FR-005.
- `match_explanation` references specific traits from the VocalProfile as
  required by FR-006.
- `confidence` distinguishes reliable matches from exploratory suggestions
  as required by FR-007.
- `reference_songs` are illustrative examples, not links to a music database.

### SongDirection (Knowledge Base)

A curated entry in the recommendation knowledge base. Not user-generated.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| id | string | PK | Stable identifier (e.g., "warm_ballad") |
| genre | string | not null | Genre label |
| sub_style | string | not null | Specific style name |
| tempo_range | JSON | not null | `{"low": 60, "high": 85}` |
| key_range | JSON | not null | `{"low": "Bb2", "high": "F4"}` |
| vocal_difficulty | integer | 1–5 | Typical vocal difficulty |
| mood_affinity | JSON | not null | Expected mood ranges (valence, energy, tension) |
| timbre_affinity | JSON | not null | Preferred timbre ranges (warmth, brightness, breathiness) |
| rhythm_requirements | JSON | not null | Stability and syncopation expectations |
| expression_traits | JSON | not null | Array of relevant expression traits |
| description | string | not null | Human-readable style description |
| example_songs | JSON | not null | Array of well-known example songs |
| explanation_templates | JSON | not null | Parameterized explanation templates |
| active | boolean | default true | Whether this direction is currently used |

**Notes**:
- This is a reference/configuration table, pre-populated by the team.
- 30–40 entries for v1, curated based on common vocal styles.
- Stored as a JSON file in the codebase for v1 (no DB needed for read-only
  reference data). Can be migrated to DB if dynamic editing is needed.

## Relationships

```
User (1) ──── (0..*) AnalysisSession
AnalysisSession (1) ──── (1) VocalProfile
AnalysisSession (1) ──── (3..8) Recommendation
SongDirection (reference, not FK-linked) ←── Recommendation.genre + sub_style
```

---

## State Transitions

### AnalysisSession.status

```
[created] → "processing" → "completed"
                         → "failed"
```

- **processing**: Audio features received, ML pipeline running.
- **completed**: VocalProfile and Recommendations generated successfully.
- **failed**: Analysis could not complete (bad audio quality, server error).
  `error_message` contains the reason.

---

## Validation Rules

- `pitch_min_hz` MUST be < `pitch_max_hz`
- `audio_duration_sec` MUST be between 5.0 and 180.0 (5 seconds to 3 minutes)
- `signal_quality_score` below 0.3 triggers a "low quality" warning (FR-009)
- All float scores (0.0–1.0 range) MUST be clamped to [0.0, 1.0]
- `Recommendation.rank` values MUST be unique within a session
- A session MUST have between 3 and 8 recommendations when status = "completed"
